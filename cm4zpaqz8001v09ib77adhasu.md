### 這是一個學習與問題解決的框架，稱為 **LEARNER 框架**，用於幫助用戶在學習或解決問題時，採取系統化的方法。每個字母代表一個步驟，構成了一個清晰的流程：

### **LEARNER Framework 解釋**：

1. **L (Learn Context) 學習上下文**
    
    * 理解問題或主題的背景和相關資料。
        
    * 問題的上下文越明確，越容易找到合適的答案。
        
2. **E (Explore Broadly) 由廣到細**
    
    * 初步廣泛探索，從大範圍內獲取相關信息。
        
    * 尋找更多細節，逐步縮小範圍。
        
3. **A (Ask Clearly) 明確問題**
    
    * 定義問題，確保提問清晰且有指向性。
        
    * 表達簡單明瞭，以便他人理解和回應。
        
4. **R (Refine Questions) 精煉問題**
    
    * 在探索過程中，針對模糊點進行更精確的提問。
        
    * 根據回饋或學到的內容調整問題方向。
        
5. **N (Need Specific Output) 要求特定輸出**
    
    * 明確想要得到的結果或答案的形式（如報告、數據、清單等）。
        
6. **E (Encourage Output) 鼓勵生成更多內容**
    
    * 鼓勵提供更多相關信息、視角或方案。
        
    * 通常用於激發創意或發掘潛在選項。
        
7. **R (Review Outcomes) 審視結果**
    
    * 反思結果，總結學到的知識和收穫。
        
    * 分析方法是否有效，並考慮如何改進。
        

### **適用場景**：

* **學習新知識**：在處理未知領域時提供結構化學習方法。
    
* **問題解決**：用於面對複雜問題的分步解決。
    
* **團隊合作**：幫助團隊成員形成有效的溝通和解決問題流程。案例 1：學習一門新技術（例如學習 Python）
    

1. **L (Learn Context) 學習上下文**
    
    * 目標：學會 Python 基本語法，並完成一個簡單的自動化腳本。
        
    * 背景：有一些程式設計經驗，但對 Python 完全陌生。
        
2. **E (Explore Broadly) 由廣到細**
    
    * 瀏覽 Python 官方文檔和教學網站，了解 Python 是什麼、主要功能和常見應用領域。
        
    * 搜尋入門教程、安裝 Python 環境的步驟。
        
3. **A (Ask Clearly) 明確問題**
    
    * 問題1：如何安裝 Python？
        
    * 問題2：Python 的基本語法是什麼？例如，如何定義變量、進行循環和條件判斷？
        
4. **R (Refine Questions) 精煉問題**
    
    * 問題1：如何在 Windows/Mac 安裝 Python 並配置環境變量？
        
    * 問題2：如何使用 Python 寫一個可以自動命名文件的腳本？
        
5. **N (Need Specific Output) 要求特定輸出**
    
    * 希望獲得一個具體的代碼片段和逐步的解釋，幫助實現自動化腳本。
        
6. **E (Encourage Output) 鼓勵生成更多內容**
    
    * 詢問 Python 社群是否有其他腳本範例或學習資源。
        
    * 激發想法：還有哪些日常任務可以用 Python 自動化？
        
7. **R (Review Outcomes) 審視結果**
    
    * 回顧學到的基礎知識，例如安裝、基本語法等。
        
    * 思考下一步：是否可以挑戰更高階的 Python 專案，如數據分析或網頁爬蟲？

### 案例 2：解決工作問題（例如提升團隊協作效率）

1. **L (Learn Context) 學習上下文**
    
    * 團隊反饋：任務分配效率低，大家對進度跟踪不清楚。
        
    * 了解當前工具和流程，例如使用電子郵件和基本表格管理任務。
        
2. **E (Explore Broadly) 由廣到細**
    
    * 搜索市面上有哪些專為團隊協作設計的工具（如 Trello、Asana、Notion）。
        
    * 比較各工具的功能、優缺點。
        
3. **A (Ask Clearly) 明確問題**
    
    * 問題1：哪種工具適合小型團隊的項目管理？
        
    * 問題2：如何快速上手並培訓團隊使用新工具？
        
4. **R (Refine Questions) 精煉問題**
    
    * 問題1：免費工具和付費工具之間的關鍵差異是什麼？
        
    * 問題2：是否有簡單的模板可直接用於我們的工作流程？
        
5. **N (Need Specific Output) 要求特定輸出**
    
    * 期望一個推薦工具的列表，以及一個初步的實施計劃（包含分工、上手教程）。
        
6. **E (Encourage Output) 鼓勵生成更多內容**
    
    * 鼓勵團隊提出其他痛點和建議。
        
    * 問自己：新工具是否可以與現有系統整合？
        
7. **R (Review Outcomes) 審視結果**
    
    * 審視選擇的工具是否提升效率，例如任務完成速度是否更快。
        
    * 總結經驗，為未來類似的工具選擇提供參考。

如何用在NotebookLM 提问？

在 NotebookLM（Google 的筆記型人工智慧助理）中，**LEARNER 框架** 可以幫助你有條理地提問和管理上下文，以便更高效地獲得答案和支持。以下是具體操作方法：

---

### **使用 LEARNER 框架 提問的步驟**

#### 1\. **L (Learn Context) 學習上下文**

* **操作**：輸入與問題相關的背景資訊，確保 NotebookLM 理解你的需求。
    
* **例子**：
    
    * "我正在學習 Python，希望使用它來自動化一些文件管理的任務。"
        
    * "我有一個專案需要分析財務數據，目前有一個 Excel 表格作為數據源。"
        

#### 2\. **E (Explore Broadly) 由廣到細**

* **操作**：開始提出一般性問題以獲取廣泛的資訊，然後逐步縮小範圍。
    
* **例子**：
    
    * "有哪些適合初學者的 Python 自動化案例？"
        
    * "如何用 Python 處理 Excel 數據？"
        

#### 3\. **A (Ask Clearly) 明確問題**

* **操作**：用簡單、清楚的語句表達你的問題，確保 NotebookLM 可以準確回答。
    
* **例子**：
    
    * "如何用 Python 讀取和重命名一個文件夾中的所有文件？"
        
    * "哪些 Python 庫適合處理財務數據，並能生成視覺化圖表？"
        

#### 4\. **R (Refine Questions) 精煉問題**

* **操作**：根據 NotebookLM 的回應，深入詢問細節，調整或重新表述問題。
    
* **例子**：
    
    * 初始提問：「如何用 Python 自動化重命名文件？」
        
    * 精煉問題：「如果文件名中包含日期，如何根據日期排序並重命名？」
        

#### 5\. **N (Need Specific Output) 要求特定輸出**

* **操作**：說明你希望 NotebookLM 的回答形式（例如代碼片段、步驟、概念解釋）。
    
* **例子**：
    
    * "請提供一段 Python 代碼，用於讀取和整理 Excel 數據。"
        
    * "給我一個簡單的教程，教如何用 Matplotlib 繪製折線圖。"
        

#### 6\. **E (Encourage Output) 鼓勵生成更多內容**

* **操作**：邀請 NotebookLM 提供多個視角或延伸的資訊。
    
* **例子**：
    
    * "除了 Pandas 庫，還有哪些工具可以高效處理大數據表？"
        
    * "是否有其他自動化方式（例如使用批處理文件或其他腳本語言）來完成相同任務？"
        

#### 7\. **R (Review Outcomes) 審視結果**

* **操作**：查看 NotebookLM 的回應，檢查是否符合你的需求，並反思結果的應用價值。
    
* **例子**：
    
    * "這段代碼是否適合處理超大文件夾中的文件？"
        
    * "這些圖表是否符合我們的報告標準？是否需要調整顏色或標籤樣式？"

### **實際例子：用 NotebookLM 解決文件管理問題**

**背景（L）**  
「我需要用 Python 將文件夾中的 PDF 文件按日期重新命名，日期資訊嵌在文件名中。」

**提問（E + A）**

1. "如何用 Python 讀取文件夾中的所有文件名？"
    
2. "如何提取文件名中的日期並根據日期排序？"
    
3. "有沒有代碼範例，能夠將文件按排序重新命名？"
    

**精煉問題（R）**  
「如果文件名中存在不同格式的日期，如何識別並統一格式？」

**輸出需求（N）**  
「請提供 Python 代碼，並附加註解解釋每一步的功能。」

**激發更多內容（E）**  
「除了重命名文件，還有哪些方法可以幫助整理大批量的 PDF 文件？」

**審視結果（R）**  
「生成的代碼是否運行良好？是否需要進一步調整以適應其他文件格式？」
